Predictive Maintenance

End-to-End ML Lifecycle on Azure ML + MLflow

AIN-3009 Delivering AI Applications with MLOps
Ayhan Gurbangeldiyev — 2020053

Dataset & Framing

  • AI4I 2020: 10,000 machine telemetry rows
  • Target: Machine failure, only 3.4% positive
  • Leakage columns dropped: TWF, HDF, PWF, OSF, RNF
  • Metrics: ROC-AUC, PR-AUC, recall, F1

Architecture

Local train + tune
  │ mlflow.set_tracking_uri(azureml://...)
  ▼
Azure ML Workspace
  ├─ Experiments / artifacts
  ├─ Model Registry
  ▼
Managed Online Endpoint → REST /score
  ▲
Monitoring metrics logged to MLflow

Training stays local for cost control; lifecycle state lives in Azure ML.

Takeaways

  • Lifecycle > model: tracking, registry, serving, monitoring

  • Azure credit used for tracking, registry, and endpoint demo
  • Endpoint can be deleted after demo to control cost
